# UNO-Q Status Web — 设计文档

**日期**: 2026-07-06
**项目**: UNO-Q 系统状态 Web App (Workshop)

---

## 1. 概述

一个运行在 Arduino UNO-Q 开发板 Linux 侧的轻量 Web 应用，在浏览器中展示 CPU、内存、磁盘（根分区 `/` 和 `/home` 分区）、系统运行时间和主机名，每 5 秒自动刷新。

**核心目标**: 通过一个可运行、可理解的 Web App，展示 UNO-Q 不只是传统开发板，也能作为 Linux 计算节点运行网络服务。

---

## 2. 架构

```
┌──────────────────────────────────────────────────────────┐
│                   同一局域网内的浏览器                      │
│          http://hostname.local:3000 或 http://IP:3000     │
└──────────────────────┬───────────────────────────────────┘
                       │ HTTP (局域网)
                       ▼
┌──────────────────────────────────────────────────────────┐
│                  UNO-Q (aarch64 Linux)                    │
│                                                          │
│  ┌────────────────────────────────────────────────────┐  │
│  │  server.js (唯一核心文件, ~120行 + 注释)             │  │
│  │                                                    │  │
│  │  Express 监听 0.0.0.0:3000                         │  │
│  │                                                    │  │
│  │  GET /            → 返回内嵌 HTML 页面              │  │
│  │  GET /api/status  → 返回系统状态 JSON               │  │
│  │                                                    │  │
│  │  数据来源 (纯 Node.js 内置模块, 零额外依赖):         │  │
│  │    os.cpus()        → CPU 使用率                    │  │
│  │    os.totalmem()    → 内存总量                      │  │
│  │    os.freemem()     → 空闲内存                      │  │
│  │    os.uptime()      → 系统运行时间 (秒)              │  │
│  │    os.hostname()    → 主机名                        │  │
│  │    child_process    → execSync('df -h / /home')     │  │
│  │                       获取两个分区的磁盘信息          │  │
│  └────────────────────────────────────────────────────┘  │
└──────────────────────────────────────────────────────────┘
```

### 关于 0.0.0.0 绑定

Express 默认只监听 `127.0.0.1`（本机回环），只有本机能访问。必须显式指定 `0.0.0.0` 才能让同一局域网的其他设备通过 IP 地址或 mDNS 域名访问 UNO-Q：

```js
// ✅ 正确 — 局域网可访问
app.listen(3000, '0.0.0.0', () => { ... });

// ❌ 错误 — 仅本机可访问
app.listen(3000, () => { ... });
```

这是 Workshop 的一个关键知识点：**0.0.0.0 = 监听所有网络接口**。

## 3. 技术选型

| 选择 | 理由 |
|------|------|
| **Node.js + Express** | Node.js 在 UNO-Q 上已预装 (v24)；Express 是 Node 生态最广泛使用的 Web 框架，`res.json()` 一行出 JSON API |
| **单文件 server.js** | 所有代码在一个文件中，打开就能从头读到尾，适合教学 |
| **内嵌 HTML 模板字符串** | 无需静态文件目录，无需额外模板引擎，降低理解门槛 |
| **os 内置模块** | 获取 CPU/内存/uptime/hostname 零依赖；`os.cpus()` 返回每个核心的使用时间，计算差值得到使用率 |
| **child_process.execSync + df** | 获取磁盘信息的唯一轻量方式，无需 `systeminformation` 这种重量级包 |
| **纯 CSS 无框架** | 页面简单，不需要引入 Bootstrap/Tailwind；内联样式 + 几行 CSS 足够 |
| **仅 1 个依赖: express** | package.json 干净到让人安心 |

### CPU 使用率计算方式

```
第一次采样: os.cpus()[*].times → 记录 (idle, total)
等待 200ms
第二次采样: os.cpus()[*].times → 记录 (idle, total)

CPU 使用率 = 100 - (idle差值 / total差值 * 100)
```

这是唯一需要解释的技巧：CPU 使用率是瞬时值，需要两次采样取差值。这比解析 `/proc/stat` 更可读。

## 5. API 设计

```
GET /api/status
Response (JSON):
{
  "cpu": 12.5,
  "memory": { "used": 234, "total": 512, "unit": "MB" },
  "disks": [
    { "mount": "/",       "used": "1.2G", "total": "7.8G", "percent": 15 },
    { "mount": "/home",   "used": "3.4G", "total": "15G",  "percent": 23 }
  ],
  "uptime": 274935,
  "hostname": "Group11-B"
}
```

- `cpu`: 0-100 的浮点数，前端显示百分比
- `memory.used/total`: 已换算为 MB 的整数
- `disks`: 数组，每个元素对应一个挂载点；由 `df -h / /home` 输出解析
- `disks[].percent`: df 命令解析出的整数百分比
- `uptime`: 秒数，由前端格式化为可读文本

## 7. 数据流 & 前端自动刷新

```
页面加载
  │
  ▼
fetch('/api/status')
  │
  ▼
更新 DOM（数值 + 进度条 + 颜色）
  │
  ▼
setInterval(fetchAndUpdate, 5000)  ← 每 5 秒
  │
  └── 异常时：保留上次数据，底部显示 "连接失败" 提示
```

前端不引入任何框架。`fetch()` 是浏览器原生 API，`document.getElementById().textContent` 更新 DOM——两个概念就讲清楚前端如何和后端通信。

## 8. 错误处理

| 场景 | 处理 |
|------|------|
| `/api/status` 内部异常（如 df 命令失败） | 返回 `{ error: "..." }`，HTTP 500；前端显示 "数据获取失败" |
| 前端 fetch 失败（网络断开） | 保留上次数据，状态栏显示 "⚠️ 连接失败，将在 5 秒后重试" |
| CPU 采样异常 | CPU 字段显示 "N/A" |
| df 命令不可用或分区不存在 | 对应磁盘字段显示 "N/A" |

## 10. 已知限制

- CPU 使用率是瞬时采样值，不是长期平均值
- 磁盘信息仅展示根分区 `/` 和 `/home` 分区，不包含其他挂载点
- 没有历史数据图表
- 仅支持 HTTP，不支持 HTTPS
